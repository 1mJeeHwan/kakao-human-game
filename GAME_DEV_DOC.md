## 프로젝트 개요

### 프로젝트명
**인간 키우기** (Human Upgrade Simulator)

### 프로젝트 설명
카카오톡 챗봇 기반의 인간 육성 게임입니다. 사용자는 골드를 소비하여 인간 캐릭터를 강화하고, **랜덤 수식어 시스템**을 통해 다양한 조합의 캐릭터를 생성합니다. 강화된 인간을 판매하여 수익을 얻는 중독성 있는 게임플레이를 제공합니다.

### 핵심 기능
- 👤 **인간 강화**: 확률 기반 레벨 강화 시스템 (성공/실패/사망)
- 🏷️ **랜덤 수식어 시스템**: 칭호(접두) + 직업(접미) 모두 랜덤 조합
- 💰 **인간 판매**: 강화된 인간을 골드로 판매 (수식어 등급에 따른 보너스)
- 📊 **통계 시스템**: 강화 시도 횟수, 성공률, 최고 달성 레벨 등
- 🏆 **랭킹 시스템** (추후 확장)

### 게임 컨셉
> "어떤 인간이 탄생할지 모른다! 랜덤 조합의 재미!"

- 칭호와 직업이 모두 랜덤으로 결정
- 수많은 조합 가능: **"게으른 회사원"**, **"천재적인 요리사"**, **"전설의 의사"**
- 희귀한 조합을 노리는 수집 요소

### 개발 기간
- **1주차**: 환경 설정 및 기본 API 개발
- **2주차**: 수식어 시스템 구현 및 카카오 오픈빌더 연동
- **3주차**: 밸런스 조정 및 QA
- **4주차**: 배포 및 모니터링

### 예산
- **개발비**: 무료 (개인 프로젝트)
- **인프라 비용**: 무료 (Render + MongoDB Atlas 무료 플랜)
- **유지보수 비용**: 유저 증가 시 월 $5~10 예상

## 시스템 아키텍처

```
[사용자]
   ↓
[카카오톡 앱]
   ↓
[카카오톡 채널]
   ↓
[카카오 i 오픈빌더] - 시나리오 관리
   ↓
[스킬서버 API] (Render.com)
   ↓
[MongoDB] (Atlas) - 유저 데이터 저장
```

### 통신 흐름
1. 사용자가 카카오톡 채널에서 메시지 입력
2. 카카오 오픈빌더가 시나리오에 따라 스킬서버 API 호출
3. 스킬서버가 MongoDB에서 유저 데이터 조회/수정
4. 스킬서버가 응답 JSON 반환
5. 오픈빌더가 응답을 카카오톡 메시지로 변환
6. 사용자가 카카오톡에서 결과 확인

## 게임 시스템 설계

### 1. 랜덤 수식어 시스템 (Modifier System)

게임의 핵심 차별화 요소입니다. **칭호(접두)**와 **직업(접미)** 모두 랜덤으로 결정됩니다.

#### 1-1. 접두 수식어 (칭호/성격) - 랜덤

캐릭터 생성 시 랜덤으로 부여됩니다.

| 등급 | 칭호 목록 | 판매 보너스 | 획득 확률 |
|------|-----------|-------------|-----------|
| **일반** | 평범한, 순수한, 착한, 소심한 | +0% | 40% |
| **고급** | 부지런한, 성실한, 밝은, 듬직한 | +10% | 30% |
| **희귀** | 용맹한, 현명한, 천재적인, 카리스마 있는 | +25% | 20% |
| **영웅** | 위대한, 고귀한, 빛나는 | +50% | 8% |
| **전설** | 전설의, 신화적인, 불멸의 | +100% | 2% |

#### 1-2. 접미 수식어 (직업) - 랜덤

캐릭터 생성 시 랜덤으로 부여됩니다. 레벨업 시 직업명 앞에 등급 수식어가 붙습니다.

**직업 카테고리**:

| 카테고리 | 직업 목록 |
|----------|-----------|
| **일반직** | 회사원, 공무원, 은행원, 영업사원, 프리랜서 |
| **전문직** | 의사, 변호사, 교수, 연구원, 회계사 |
| **기술직** | 개발자, 엔지니어, 디자이너, 건축가, 요리사 |
| **예체능** | 가수, 배우, 화가, 운동선수, 작가 |
| **판타지** | 마법사, 전사, 기사, 궁수, 도적 |
| **특수직** | 탐정, 용병, 연금술사, 모험가, 사냥꾼 |

**직업 등급 (확률)**:

| 등급 | 획득 확률 | 판매 보너스 |
|------|-----------|-------------|
| 일반 | 50% | +0% |
| 고급 | 30% | +15% |
| 희귀 | 15% | +30% |
| 전설 | 5% | +60% |

#### 1-3. 레벨별 직업 수식어

레벨에 따라 직업명 앞에 등급이 붙습니다.

| 레벨 | 등급 수식어 | 예시 (요리사) |
|------|-------------|---------------|
| 0 | 수습 | 수습 요리사 |
| 1~3 | 견습 | 견습 요리사 |
| 4~6 | (없음) | 요리사 |
| 7~9 | 숙련 | 숙련 요리사 |
| 10~12 | 베테랑 | 베테랑 요리사 |
| 13~14 | 마스터 | 마스터 요리사 |
| 15 | 그랜드마스터 | 그랜드마스터 요리사 |

#### 1-4. 수식어 조합 예시

```
"평범한 +0 수습 회사원"           → 가장 흔한 조합
"부지런한 +5 개발자"              → 고급 칭호 + 기술직
"천재적인 +10 베테랑 의사"        → 희귀 칭호 + 전문직
"전설의 +15 그랜드마스터 마법사"  → 최고 조합
"게으른 +7 숙련 용병"             → 재미있는 조합
"위대한 +12 베테랑 요리사"        → 영웅 칭호 + 기술직
```

#### 1-5. 총 조합 수

- 칭호: 약 15종
- 직업: 약 30종
- 레벨 등급: 7단계
- **가능한 조합: 15 × 30 × 7 = 3,150가지**

### 2. 강화 시스템 (Upgrade System)

#### 2-1. 기본 규칙

- **강화 대상**: 인간 캐릭터의 레벨 (0~15)
- **강화 비용**: 레벨에 따라 증가
- **강화 결과**: 성공 / 실패(유지) / 사망(초기화)

#### 2-2. 강화 결과

| 결과 | 설명 | 효과 |
|------|------|------|
| **성공** | 강화 성공! | 레벨 +1, 직업 등급 변경 가능 |
| **실패** | 강화 실패 | 레벨 유지, 골드만 소비 |
| **사망** | 인간이 사망... | 레벨 0 초기화, 새 캐릭터 생성 |

#### 2-3. 사망 시 처리

- 캐릭터 완전 초기화
- **새로운 랜덤 칭호 + 직업 부여**
- 통계에 사망 횟수 기록
- "다음에는 어떤 인간이 올까?" 기대감 유발

---

### 3. 판매 시스템 (Sell System)

#### 3-1. 기본 판매가 계산

```javascript
baseSellPrice = 2^level * 1000
```

| 레벨 | 기본 판매가 |
|------|-------------|
| 0 | 0G (판매 불가) |
| 1 | 2,000G |
| 5 | 32,000G |
| 10 | 1,024,000G |
| 15 | 32,768,000G |

#### 3-2. 수식어 보너스 적용

```javascript
titleBonus = 칭호 등급 보너스 (0% ~ 100%)
jobBonus = 직업 등급 보너스 (0% ~ 60%)
totalBonus = titleBonus + jobBonus

finalSellPrice = baseSellPrice * (1 + totalBonus)
```

**예시 계산**:
| 캐릭터 | 기본가 | 칭호 보너스 | 직업 보너스 | 총 판매가 |
|--------|--------|-------------|-------------|-----------|
| 평범한 +10 회사원 | 1,024,000G | +0% | +0% | 1,024,000G |
| 현명한 +10 의사 | 1,024,000G | +25% | +30% | 1,587,200G |
| 전설의 +10 마법사 | 1,024,000G | +100% | +60% | 2,662,400G |
| 전설의 +15 그랜드마스터 마법사 | 32,768,000G | +100% | +60% | 85,196,800G |

#### 3-3. 판매 시 처리

- 골드 획득
- 캐릭터 완전 초기화 (새 랜덤 칭호 + 직업)
- 통계 업데이트

---

### 4. 수식어 리롤 시스템 (Reroll System)

#### 4-1. 칭호 리롤

현재 칭호가 마음에 들지 않을 때 재설정합니다.

```javascript
titleRerollCost = 3000 + (level * 500)
```

| 레벨 | 칭호 리롤 비용 |
|------|----------------|
| 0 | 3,000G |
| 5 | 5,500G |
| 10 | 8,000G |
| 15 | 10,500G |

#### 4-2. 직업 리롤

현재 직업이 마음에 들지 않을 때 재설정합니다.

```javascript
jobRerollCost = 5000 + (level * 1000)
```

| 레벨 | 직업 리롤 비용 |
|------|----------------|
| 0 | 5,000G |
| 5 | 10,000G |
| 10 | 15,000G |
| 15 | 20,000G |

#### 4-3. 리롤 규칙

- 칭호/직업 각각 독립적으로 리롤 가능
- 레벨은 유지
- 확률 테이블에 따라 새로운 수식어 부여

## 데이터베이스 설계

### MongoDB Collection: `users`

```javascript
{
  _id: ObjectId("..."),
  kakaoUserId: "1234567",                  // 카카오 유저 고유 ID

  // 인간 캐릭터 정보
  human: {
    level: 0,                              // 강화 레벨 (0~15)

    // 칭호 (접두 수식어)
    title: {
      name: "평범한",                       // 칭호 이름
      grade: "common",                     // 등급
      bonusRate: 0                         // 판매 보너스 (0.0 ~ 1.0)
    },

    // 직업 (접미 수식어)
    job: {
      name: "회사원",                       // 직업 이름
      category: "office",                  // 카테고리
      grade: "common",                     // 등급
      bonusRate: 0                         // 판매 보너스 (0.0 ~ 0.6)
    },

    // 레벨에 따른 직업 등급 (자동 계산)
    jobPrefix: "수습"                       // 수습/견습/숙련/베테랑/마스터/그랜드마스터
  },

  // 재화
  gold: 10000,

  // 통계
  stats: {
    totalAttempts: 0,                      // 총 강화 시도 횟수
    successCount: 0,                       // 성공 횟수
    failCount: 0,                          // 실패 횟수
    deathCount: 0,                         // 사망 횟수
    maxLevel: 0,                           // 최고 달성 레벨
    totalGoldEarned: 0,                    // 총 획득 골드
    totalGoldSpent: 0,                     // 총 소비 골드
    totalTitleRerolls: 0,                  // 칭호 리롤 횟수
    totalJobRerolls: 0,                    // 직업 리롤 횟수
    legendaryTitleCount: 0,                // 전설 칭호 획득 횟수
    legendaryJobCount: 0                   // 전설 직업 획득 횟수
  },

  // 메타 정보
  createdAt: ISODate("..."),
  lastPlayedAt: ISODate("...")
}
```

### 인덱스 설계
```javascript
db.users.createIndex({ "kakaoUserId": 1 }, { unique: true });
db.users.createIndex({ "stats.maxLevel": -1 });
db.users.createIndex({ "human.level": -1 });
```

### 판매 가격 공식

```javascript
function getFullName(human) {
  const jobPrefix = getJobPrefix(human.level); // 수습/견습/숙련 등
  return `${human.title.name} +${human.level} ${jobPrefix} ${human.job.name}`;
}

function getSellPrice(human) {
  if (human.level === 0) return 0;

  const basePrice = Math.pow(2, human.level) * 1000;
  const titleBonus = human.title.bonusRate;  // 0 ~ 1.0
  const jobBonus = human.job.bonusRate;      // 0 ~ 0.6
  const totalBonus = titleBonus + jobBonus;

  return Math.floor(basePrice * (1 + totalBonus));
}
```
